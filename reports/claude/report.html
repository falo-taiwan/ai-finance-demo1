一、平台與架構：兩外掛同源分析

兩個資料頁皆為同一 WordPress 站的自製外掛，REST 路由骨架與 HTML 命名慣例高度一致，極可能同作者、同一套程式範本衍生。差異源自資料量級（13,238 vs 213）。

維度cae-tracker（mops-audit）ia_course_list（course-scraper）
資料主題稽核主管異動（爬 MOPS）內稽課程（爬5大培訓機構）
儲存自訂資料表（非 CPT）自訂資料表（非 CPT）
REST 骨架import/clear/status/records/record-idsimport/clear/status/courses/course-ids + log/snapshot
HTML classmops-audit-{container,filters,list,table,stats}course-{container,filters,list,table,stats}
前端篩選伺服器端 GET（JS 留空 256B）客戶端 JS 篩選（已全載）
顯示上限每查 88 筆硬上限無上限
資料 REST 端點/records 401 鎖死/courses 公開
外掛版本1.9.82.0.0

結論：course-scraper 是「小資料、全公開」原型；mops-audit 是被資料量逼出安全強化（上限＋鎖 REST）的進階版。

二、突破上限測試（cae-tracker）

手法結果判定
前台分頁/limit 參數（per_page / limit / posts_per_page / nopaging / ma_limit / count …11 種）一律回 88 筆無效·伺服器端硬寫死
前端 JS 繞過JS 僅 256B 留空無 AJAX 可繞
paged=2 偏移分頁回 0 筆不支援 offset
REST /records（rest_route / X-WP-Nonce / _wpnonce / per_page）全部 401 rest_forbidden權限正確上鎖
資料是否為 CPT（wp/v2 曝露）types 無 mops/audit自訂表·觸及不到
/status 端點200 公開洩漏總數/版本

唯一可行路徑：透過公開搜尋表單「分格查詢」再拼接，並用本地解析（最省 token）。
